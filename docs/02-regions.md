# Районы мониторинга

## Принцип выбора районов

Ленинградская область неоднородна по погоде, рельефу, удаленности от Финского залива и Ладоги, а также по характеру малых рек. Поэтому дашборд строится не как единая область, а как набор направлений с отдельными погодными и природными профилями.

В MVP каждый район представлен одной опорной точкой. Это упрощает получение данных из погодных и астрономических источников. В следующих версиях у района может быть несколько точек: основная точка погоды, ближайшая гидрологическая точка, набор ручьев, пользовательские рыболовные места.

## Требования к данным района

Для каждого района нужно хранить:

- уникальный ID;
- человекочитаемое название;
- направление;
- опорный населенный пункт;
- широту и долготу;
- краткое описание природного профиля;
- список потенциальных водотоков или зон;
- активность района в интерфейсе;
- приоритет отображения.

## Координаты

Координаты внесены в `data/regions.json` и используются как опорные точки для погодного адаптера.

Следующий рабочий процесс:

1. проверить, что погодный API возвращает данные по каждой точке;
2. сравнить погодные различия между 5 районами;
3. при необходимости сдвинуть точку от населенного пункта к более релевантной природной зоне;
4. позже добавить несколько точек внутри района, если один населенный пункт будет слишком грубо описывать район.

## Будущая детализация

Для каждого района позже можно добавить:

- типичные ручьи и малые реки;
- ближайшие крупные водоемы, влияющие на микроклимат;
- примечания по сезонной доступности;
- оценку антропогенной нагрузки;
- типичные риски: паводок, пересыхание, мутность, запреты, трудная доступность.
